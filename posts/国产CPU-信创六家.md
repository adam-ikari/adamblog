# 国产 CPU 详解：信创六家

## 前言

信创基础设施这一层，国产 CPU 演成了六条路线。这一篇就把这六条路走一遍。

国产 CPU 这事，外行看热闹容易陷入两个极端。一个极端是"全是套壳，自主是假的"——理由是兆芯海光靠 x86 授权、鲲鹏飞腾靠 ARM 授权，似乎都不算"真自主"。另一个极端是"六家皆自主可控"——把厂商宣传话术照单全收。两个极端都没说到点上。六家 CPU 的自主程度确实差异巨大，但这种差异不是"自主"与"套壳"的二分，而是一条从"完全自研指令集"到"合资拿现成设计"的连续光谱。龙芯和申威在一头，兆芯和海光在另一头，鲲鹏和飞腾居中。理解这条光谱，比争论"算不算自主"有意义得多。

本篇逐一拆解六家：龙芯（LoongArch，中科院，完全自研指令集）、鲲鹏（ARM，华为）、飞腾（ARM，中国电子/国防科大）、兆芯（x86，威盛授权）、海光（x86，AMD 合资）、申威（SW64，自主，超算）。每家讲架构、来历、定位、自主程度，最后用一张对比表收束。

需要先框个边界：这里讲的是进了信创采购盘子的六家。信创之外还有一批走消费电子、开源路线的新兴国产 CPU（香山、此芯、瑞芯微、全志等），衡量尺度跟这六家不同，另见 [国产 CPU 新兴势力](/posts/国产CPU-新兴势力)，本篇不展开。

## 一、龙芯：唯一走通完全自研指令集的路

龙芯是六家里自主程度最高的一家，也是争议最值得讲清的一家。

它的来历很硬核：由中国科学院计算技术研究所设计，公司主体是 2010 年成立的龙芯中科[^loongson_wiki]。龙芯一号 2002 年 9 月发布，整体性能低于 Intel 奔腾 II——这个起点谈不上强，但意义在于它是国产通用 CPU 从零起步的标志（[来源](https://zh.wikipedia.org/wiki/龙芯)）。龙芯的生日定在 2001 年 8 月 19 日，那天龙芯一号的逻辑设计在 FPGA 平台上跑通。

龙芯最关键的技术选择，是指令集的演进路线。它早期用 MIPS 指令集，由 MIPS 科技公司授权，走过 MIPS II → MIPS III → MIPS64 → 自研扩展 LoongISA 的过程。但 MIPS 是别人的东西，生态也在萎缩，龙芯最终在 2021 年推出了完全自研的 LoongArch 指令集。首款采用 LoongArch 的是龙芯 3A5000，其核心在文档中被从 GS464 重命名为 LA464。到 2023 年的龙芯 3A6000，采用 LA664 微架构，仍基于 LoongArch。

LoongArch 是六家 CPU 里唯一完全脱离国际主流指令集（x86、ARM）的自研架构。这意味着两件事：一是自主程度最高，指令集的命脉攥在自己手里，不受任何外资授权制约；二是生态最孤立，x86 和 ARM 有现成的庞大软件生态，LoongArch 得自己一点点攒，操作系统、编译器、应用都得专门适配。龙芯支持 Linux（Debian、Deepin、UOS 等）、BSD 和多种 RTOS，开发者持续向 Linux 主线内核提交代码，但生态体量与 x86/ARM 不可同日而语。

这是完全自研指令集的必然代价。龙芯选了一条最难的路：放弃了现成生态的便利，换来了真正的指令集自主。它能不能走通，取决于 LoongArch 生态能否壮大到支撑实际使用——这是悬而未决的问题，但至少方向上它是最"纯粹"的自主。

## 二、鲲鹏：华为的 ARM 自研核心

鲲鹏是六家里出身最"商业"的一家，背后是华为的海思（HiSilicon）。

鲲鹏 920 是代表产品，原名 Hi1620，是海思第四代服务器处理器，2018 年宣布、2019 年正式上市，采用台积电 7nm HPC 工艺[^kunpeng_wiki]。它实现 ARMv8.2-A 指令集，华为称其核心支持"几乎全部 ARMv8.4-A 特性（少数除外）"（[来源](https://en.wikipedia.org/wiki/Kunpeng_920)）。这里有个细节值得注意：鲲鹏用的不是 ARM 公版核心，而是华为自研的 TaiShan V110 核心，32 至 64 核，主频最高 2.6 GHz，4 路超标量、乱序执行（[来源](https://en.wikipedia.org/wiki/Kunpeng_920)）。

"自研核心 + ARM 架构授权"是鲲鹏的路线。它拿的是 ARM 指令集架构授权（architecture license），不是直接买 ARM 公版核心设计，核心微架构自己做的。这让它比直接用公版的厂商自主程度高一层，但仍未脱离 ARM 生态，指令集仍是 ARM 的，软件生态借的是 ARM 的。鲲鹏用于华为的 TaiShan 2280 V2、5280 V2、XA320 V2 等服务器。

鲲鹏的定位很明确：服务器。它不走桌面消费市场，主攻数据中心和政企服务器。华为的算力栈里，鲲鹏是硬件底座，配上欧拉操作系统（openEuler）和华为云，构成一整套国产服务器方案。但鲲鹏也面临 ARM 服务器生态的通病——x86 仍是服务器软件的主流，ARM 服务器需要大量适配。华为的体量和投入能部分弥补这一点，但 ARM 路线在服务器市场的天花板，是鲲鹏绕不过的结构性约束。

## 三、飞腾：CEC 与国防科大的 ARM 路

飞腾的来历和鲲鹏同属 ARM 路线，但出身不同，它背后是中国电子（CEC）和国防科技大学。

飞腾的公司主体是天津飞腾信息技术，研发渊源可追溯到国防科技大学。它走 ARM 架构授权路线，和鲲鹏一样是"借 ARM 生态、保一定自主"。飞腾官网将其产品线分为三大系列：高性能服务器 CPU（腾云 S 系列，如腾云 S5000C、S2500、FT-2000+/64）、高效能桌面 CPU（腾锐 D 系列，如腾锐 D3000、D2000、FT-2000/4）、高端嵌入式 CPU（腾珑 E 系列，如腾珑 E2000）。从服务器到桌面到嵌入式，覆盖比鲲鹏更全。

飞腾的定位比鲲鹏更"信创"。它深耕党政军市场，官网自称联手 8500 多家产业生态伙伴，适配 15 万余款软件应用，行业覆盖党政、电信、金融、电力、能源、交通、教育。这个生态体量是政策适配堆出来的，和鲲鹏依托华为商业体系不同，飞腾更靠政企采购驱动，是中国电子信创全栈里的 CPU 那一环。

::: tip 关于飞腾的核验
飞腾官网（phytium.com.cn）确认了其三大产品线与 ARM 路线、政企市场定位。其与国防科技大学、中国电子（CEC）的资本与研发渊源，维基百科尚无独立的飞腾处理器条目可资直接佐证，此处依公开行业资料与官网口径，详细股权与沿革以官方为准。
:::

飞腾与鲲鹏并列为 ARM 路线的两大代表，但二者靠山不同：鲲鹏背靠华为的商业与研发体系，飞腾背靠 CEC 的国资信创体系。同样是 ARM 授权，背后的产业逻辑不一样。

## 四、兆芯：威盛授权来的 x86

兆芯是 x86 路线之一，自主程度比龙芯低不少，但兼容性是六家里最好的之一。

兆芯的公司主体是上海兆芯集成电路有限公司，2013 年成立，总部上海张江，国资控股，上海市国资委下属单位持股约 80%，威盛电子（VIA）持有剩余股份。注意这个股权结构：中资控股满足信创的 25% 红线，但威盛（台湾）持股，技术渊源也来自威盛。兆芯是 fabless（无厂半导体）公司。

兆芯的 x86 授权不是 Intel 直接给的，而是来自威盛。Intel 未直接向兆芯授权 x86 架构，兆芯的 x86 能力继承自威盛早年从 Intel 获得的授权，初期 CPU 基于 VIA Nano 架构改良。也就是说，兆芯的 x86 是"二手授权"——授权链是 Intel → 威盛 → 兆芯。这条路能跑 x86 生态，但命脉多了一层：既要看威盛，又要受 x86 授权的复杂历史约束。

产品上，兆芯有开先 KX-5000（代号"五道口"，28nm，中国首颗支持 DDR4 的通用处理器）、开先 KX-6000（代号"陆家嘴"，16nm，CPU/GPU/芯片组集成，主频最高 3.0GHz）、开胜 KH-20000（服务器处理器）。官方称 KX-6000 多核性能接近第七代 Intel Core i5，但单核性能较弱——"多核追平、单核落后"是国产 x86 的普遍特征，兆芯也不例外。

兆芯的定位是桌面和服务器，在党政军市场有收获，与联想、清华同方等整机厂商合作。它的最大优势是 x86 兼容，能直接跑 Windows 和现成 x86 软件，迁移成本低；最大短板是性能单核偏弱、授权链复杂。对需要平滑迁移既有 x86 应用的政企场景，兆芯是个务实选择，但自主程度在六家里偏后。

## 五、海光：AMD 合资的 x86 与断供之痛

海光的故事最能说明"合资换技术"的代价，是六家里自主程度最受争议的一家。

海光最初 2014 年 10 月成立，原名天津海光先进技术投资有限公司，后更名 Hygon。2016 年，基于 AMD 与中方的合资协议，成立了两家合资企业：成都海光微电子（HM，AMD 持股 51%、海光 49%）和成都海光集成电路设计（HICD，AMD 持股 30%、海光 70%）。HM 与 AMD 合作设计并由 GlobalFoundries 代工，HICD 与 HM 联合设计封装后销往中国市场。AMD 从这笔授权中获得 2.93 亿美元授权费。

海光的首款处理器 Hygon Dhyana 基于 AMD Zen 1 架构，2018 年进入中国市场。说白了，海光早期的产品就是 AMD Zen 1 的中国版，技术几乎现成拿来，自主程度在六家里最低。但这笔交易的"划算"与否，很快被现实改写。

2019 年 6 月 24 日，海光及相关实体被列入美国商务部实体清单。后果很直接：AMD 无法再向合资企业授权新的 IP，HM 也无法再从 GlobalFoundries 订购晶圆。海光只能使用已被授权的原始 Zen 1 IP 自主管理、自主迭代，新架构拿不到了，老本得自己啃。这就是"合资换技术"的底牌：技术断供的那一刻，自主迭代能力有多大，决定了此后能走多远。

尽管受限，海光此后仍自主开发产品，2022 年 8 月在科创板上市，募资约 106 亿元，是当年中国最大 IPO 之一；控股股东是曙光（Sugon），持股约 28%。海光的定位是服务器，主攻政企和数据中心，与兆芯的桌面侧重互补。

海光的故事是信创外资约束的一个标本：合资能快速换来先进技术（Zen 1 当年确属先进），但 25% 红线要求资本中资主导、实体清单又掐断后续授权，最终逼着厂商从"合资拿现成"转向"自主啃老本"。这条路起点高、后劲存疑，自主程度在六家里最薄，但 x86 兼容性和早期性能是它的硬通货。

## 六、申威：超算与自主 SW64

申威是六家里最"小众"也最特殊的一家，定位和其余五家都不一样。

申威由江南计算技术研究所（位于无锡）研发，维基记载其"主要为中国军方使用而开发"。它走自研 SW64（ShenWei-64）指令集路线，和龙芯的 LoongArch 一样属于完全自研指令集，但生态和应用场景完全不同，申威不为通用桌面或服务器，而是为超算而生。

申威与 Alpha 的关系，是容易被讲错的一点。坊间说法称早期申威微架构"受 DEC Alpha 启发"，第三代 SW-3（SW1600）被认为尤其基于 Alpha 21164。但这一说法来源是论坛和博客，可靠性存疑。而对于第四代 SW26010，超算专家 Jack Dongarra 明确指出其 SW-64 指令集"与 DEC Alpha 指令集无关"。所以准确的说法是：申威早期可能与 Alpha 有渊源（未证实），但到 SW26010 这一代，指令集已是独立自研，与 Alpha 无关。把"申威源自 Alpha"讲成定论，是不严谨的。

SW26010 是申威的代表作，2016 年问世，64 位 RISC 众核架构，4 个集群各含 64 个计算核加 1 个管理核，片上网络互联。它最著名的应用是神威·太湖之光超级计算机——这台超算曾登顶 TOP500 世界第一，是纯国产 CPU 撑起的世界级超算，也是申威自主路线最有力的证明。

申威的发展经历了四代：SW-1（2006，单核 900MHz）、SW-2（2008，双核）、SW-3/SW1600（2010，16 核 64 位）、SW26010（2016，众核）。它不参与通用 CPU 的市场竞争，在桌面和服务器市场几乎看不到申威，但超算和特定高算场景里，它是完全自主指令集能扛起世界级算力的孤例。自主程度高、生态极窄、定位专精，是申威的真实面貌。

## 七、对比：六条路线一张表

六家并置，自主程度与定位的分化便一目了然。

| 厂商 | 架构/指令集 | 来历/授权 | 自主程度 | 定位 | 代表产品 |
|---|---|---|---|---|---|
| 龙芯 | LoongArch（自研） | 中科院计算所，龙芯中科 | 高（完全自研指令集） | 桌面/服务器/工控 | 3A5000、3A6000 |
| 鲲鹏 | ARMv8.2-A | 华为海思，自研 TaiShan 核心 | 中（ARM 架构授权） | 服务器 | 鲲鹏 920 |
| 飞腾 | ARM | 国防科大/中国电子（CEC），天津飞腾 | 中（ARM 架构授权） | 服务器/桌面/嵌入式 | 腾云 S5000C、腾锐 D3000 |
| 兆芯 | x86/x86-64 | 上海兆芯，x86 授权经威盛（VIA） | 偏低（二手 x86 授权） | 桌面/服务器 | KX-6000、KH-20000 |
| 海光 | x86（兼容） | 与 AMD 合资，基于 Zen 1 | 低（合资拿现成，已断供） | 服务器 | Hygon Dhyana |
| 申威 | SW64（自研） | 江南计算技术研究所（无锡） | 高（完全自研指令集） | 超算/高算 | SW26010 |

这张表里有几条值得拎出来的线索。

自主程度呈两极加中间的分布。龙芯、申威在"完全自研指令集"一极，命脉完全在手，代价是生态孤立；兆芯、海光在"授权/合资"另一极，x86 兼容性好、性能起点高，代价是命脉受制于人；鲲鹏、飞腾居中，借 ARM 架构授权、自研核心，在自主和生态间取平衡。没有一条是完美解，各自在"自主"和"可用"之间选了不同的点。

定位也呈分化。鲲鹏、海光、飞腾的服务器侧重，对应政企和数据中心替换；兆芯的桌面侧重，对应终端办公替换；龙芯横跨桌面服务器工控，靠自研生态铺；申威独走超算，不参与通用市场竞争。这种分化不是厂商各凭喜好的结果，而是信创全栈对不同场景的需求倒逼出来的，服务器要高性能，桌面要兼容迁移，超算要极致算力，于是六家各占一席。

授权风险是另一条暗线。海光 2019 年被列实体清单后断供、兆芯的 x86 授权链横跨威盛，都说明授权路线的脆弱。龙芯从 MIPS 转 LoongArch，正是吸取了"靠别人指令集不踏实"的教训。申威与 Alpha 的关系从"可能渊源"到"明确无关"，也折射出摆脱外来架构痕迹的过程。自主可控这四个字，在 CPU 这层不是口号，而是用断供风险和生态代价换来的具体选择。

## 八、客观看待：没有一条路是轻松的

六家 CPU 走完，一个总印象是：国产 CPU 确实从无到有立起来了，但每条路都背着各自的包袱，没有一条是轻松的。

完全自研的龙芯和申威，自主程度最高，但龙芯要独自撑起一个新指令集生态，申威困在超算窄场景，自主的代价是孤独。授权路线的鲲鹏飞腾，借了 ARM 生态的力，但 ARM 服务器在 x86 主导的服务器市场本就难打，且架构授权仍受 ARM 制约。合资与二手授权的兆芯海光，兼容性最好、性能起点最高，可一旦外部环境生变（海光已是活例），后续迭代就成问题。

这意味着，拿"自主"或"套壳"的二分法去评判六家，是粗暴且失真的。真实的图景是一条连续光谱：从龙芯申威的完全自研，到鲲鹏飞腾的架构授权自研核心，再到兆芯的二手 x86 授权、海光的合资拿现成，自主程度逐级递减，兼容性逐级递增。每一档都是"自主"与"可用"之间的一次具体妥协，没有哪一档天然优于另一档，要看场景需要什么。

对信创采购而言，选哪条路往往不是技术最优解，而是约束条件下的权衡：既要满足自主可控的政策要求，又要保证业务系统能跑起来，还得考虑断供风险。党政办公终端可能选兆芯（兼容迁移）、飞腾（信创栈完整）；数据中心可能选鲲鹏、海光（服务器性能）；超算选申威；追求极致自主的特定场景选龙芯。六家并存，本身就是不同约束下没有统一解的体现。

最后说句实在的：国产 CPU 这十年进步是实打实的，龙芯从奔腾 II 水平追到 3A6000 可用、鲲鹏 920 把 ARM 服务器做到业界前列、申威撑起过世界第一超算——这些都是硬成绩。但和国际顶尖（Intel、AMD、ARM 最新架构）相比，单核性能、制程工艺、生态规模仍有差距。承认差距不丢人，夸大自主才误导。六条路线的意义在于，它们让中国在 CPU 这层有了不止一个备选、不止一条退路，哪怕每条都不完美，但不再只有一条被人卡脖子的路可走。这或许才是"自主可控"在 CPU 这层最朴素的含义。

---

信创六家之外，国产 CPU 这盘棋还有另一面：一批走开源、走消费电子路线的新兴厂商，未必进得了信创名录，却把国产芯片的覆盖面铺得更宽。那是另一套衡量尺度，见 [国产 CPU 新兴势力](/posts/国产CPU-新兴势力)。
