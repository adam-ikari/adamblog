# 信创中间件与云平台调查报告

## 前言

操作系统、数据库这两层，公众多少有些印象，麒麟、达梦这些名字至少听过。可一旦说到中间件，多数人愣住：这层到底是干什么的？这种陌生感恰恰是中间件的真实处境。它处在操作系统、数据库之上，业务应用之下，把消息传递、事务调度、应用运行这些" plumbing"活儿揽过去，让上层应用不必从零造轮子。听起来重要，却天然不吸睛。

信创这层也没躲开替代的命运。过去政企核心系统的中间件，基本被两家把持：IBM 的 WebSphere 和 Oracle 的 WebLogic，两个 Java EE 应用服务器。去 IOE 的口号喊了十几年，中间件这一环的国产替代走得比想象中扎实——不是喊口号，是真的在中移动、工行的核心系统里把 IBM、Oracle 的东西换了下来。这背后是一批做了二十多年的国产中间件厂商。

本报告梳理这一层：国产中间件从哪来、各自什么定位，以及它们之上那一层，信创云平台，是什么格局。

## 一、中间件是什么，为什么单独成一层

先把这个词说清楚，否则后面都悬空。

中间件不是某一款软件，而是一类"承上启下"的基础软件。最典型的是应用服务器（Application Server）：你写了一套 Java 业务代码，不能直接丢给操作系统跑，需要一个容器来管它的生命周期、数据库连接、事务、并发、HTTP 请求——这个容器就是应用服务器。WebSphere、WebLogic、开源的 Tomcat/WildFly，都属于这一类。再往外延展，还有消息中间件（系统间可靠传话）、交易中间件（管高频事务）、数据总线（服务集成）等等。

这层之所以单独存在，是因为它把通用功能下沉，避免每个业务系统各写一套。一个银行的核心系统，应用服务器层要扛住亿级并发、保证事务不丢——这层稳不稳，直接决定业务能不能跑。也正因如此，当年 IBM、Oracle 能在这一层卖得又贵又稳，政企客户图省心，一用就是十几年。

替代这层的难度，不在功能对不对，而在"承压"。国产中间件要接的，往往是金融、电信里跑了几十年的核心账务系统，错一笔账就是事故。所以这一层的国产化，比办公软件慢、比操作系统隐蔽，但一旦替进去，含金量也最高。

## 二、东方通：中间件里的老字号

国产中间件绕不开东方通。它 1997 年成立，那年就出了第一款消息中间件 TongLINK 和交易中间件 TongEASY，2000 年又推出应用服务器 TongWeb。把这个时间线放到当时看会很感慨：九十年代末国内基础软件几乎一片空白，东方通是少数从那时就扎根中间件的。2014 年它在深交所挂牌，自称"第一家中间件 A 股上市公司"。

东方通的产品线铺得很全。基础类有 TongWeb（应用服务器，符合 Java EE/Jakarta EE 规范）、TongLINK/Q（消息中间件）、TongHttpServer；数据类有 TongESB（服务总线）、TongDXP（数据集成）；近年又往云原生和 AI 上靠，出了 TongEASY（云原生事务）和 TongAgentPlatform（智能体平台）。一个老牌中间件厂商，产品名跟着时代换了一茬又一茬，从"消息总线"到"智能体"，但底层那条"管应用运行、管系统间通信"的主线没变。

东方通在信创里的角色，说白了就是国产中间件里资历最深的那一个。它的官网上没有直接挂"替代 WebSphere"的标语，但 TongWeb 作为符合 Java EE 标准的应用服务器，功能上对标的就是 WebSphere/WebLogic。它的落地案例多在金融、政务的国产化项目里——这种项目要求全栈国产，中间件这一环自然落在它头上。资历深是优势，也是包袱：老客户多，但产品线摊得宽，每条都得维护。

## 三、宝兰德：替进核心系统的那家

如果说东方通是资历最深的，宝兰德则是"真替进了核心系统"说得最硬气的一家。

宝兰德（北京宝兰德软件股份有限公司）大约 2008 年起做中间件，2019 年在科创板上市，股票代码 688058。它的旗舰产品是 BES Application Server，应用服务器；之外还有 BES MQ（消息）、BES VBroker（分布式交易）等一整套。定位写得很直白：国产基础软件供应商。

关键的一笔在它的官网表述里：宝兰德"在中国移动 20 多个省公司的核心业务系统 CRM 和 BOSS 中替换 IBM 和 Oracle 中间件，承载服务数 10 万+，用户规模 6 亿+，国内仅此一家"。这句话信息量很大。中移动的 CRM（客户关系管理）和 BOSS（业务运营支撑系统）是电信运营商最核心的两套系统，跑的是几亿用户的计费、出账，把这种系统底下的 WebSphere/WebLogic 换成国产中间件，且"国内仅此一家"，这是中间件国产替代里少有的硬案例。Oracle 的中间件即 WebLogic，所以"替换 Oracle 中间件"就是替 WebLogic，"替换 IBM 中间件"就是替 WebSphere。

宝兰德能替进去，靠的不是泛泛宣传，而是在电信行业里深耕了十几年、把稳定性磨出来了。中间件替代最怕的就是"功能差不多但承不住压"，宝兰德的打法是选一个行业（电信）扎到底，把最难的系统替通，再往金融、证券、政府复制。这条路径比"什么都做一点"靠谱得多。它的客户名单里有工行、招行、交行、国泰君安，都是从电信往金融复制的痕迹。

## 四、普元：中间件之外的低代码路线

第三家值得一提的不是传统应用服务器厂商，而是走了另一条路的普元。

普元信息（Primeton）在科创板上市，代码 688118。它自称"全栈中间件领导者"，产品里确有应用服务器、消息中间件、企业服务总线（ESB）、iPaaS 这些中间件品类，在"新兴中间件"这一档，普元在行业报告里排领导者位置。

但普元的重心其实已经偏向低代码和数据治理。它是"国内最早的低代码实践者"之一，低代码平台内置 AI Copilot，主推企业用低代码搭应用；数据治理那块，它被中国企业数字化联盟认证为"央国企数据治理市场占有率第一"。这两块业务跟传统中间件的关系是：中间件管运行，低代码管开发，数据治理管数据底座，普元想把企业级基础软件从运行到开发到数据一条龙包下来。

把普元放进中间件这篇里，是因为它代表了这层的一个走向：纯做应用服务器，天花板就那么高；往低代码和数据治理延伸，是把中间件的能力往上长。东方通往 AI 智能体长，普元往低代码长，都是同一个逻辑，基础软件厂商不甘心只做底层管道工，想离业务更近一点。

## 五、三家的格局，以及和 WebSphere/WebLogic 的真实距离

把三家并置，国产中间件的格局就清楚了。

东方通资历最深、产品线最全，是"老牌龙头"；宝兰德把电信核心系统替通了，是"硬案例担当"；普元从中间件往低代码和数据治理延伸，是"基础软件平台化"的代表。三家上市路径也不同：东方通深交所主板（2014，最早），宝兰德、普元都走科创板（2019、随后）——科创板对"国产基础软件"这类标的友好，这也解释了为什么这批厂商集中在近几年上市。

::: tip 关于 WebSphere/WebLogic
这两者是国外应用服务器的代表。WebSphere 属 IBM，WebLogic 属 Oracle（自收购 BEA Systems 起）。二者长期占据国内政企 Java EE 中间件市场，是"去 IOE"里"E"之外、同样要被替代的一环。国产应用服务器（东方通 TongWeb、宝兰德 BES 等）功能上对标它们，落地则集中在金融、电信、政务的信创项目里。
:::

但要老实说一句：国产中间件和 WebSphere/WebLogic 的"距离"，不是功能差距，而是生态与惯性的距离。WebSphere/WebLogic 背后是几十年的企业级生态、海量经过验证的集成方案、一遇问题就有 Oracle/IBM 工程师兜底的安心感。国产中间件功能上能跑通 Java EE 规范，但要让客户敢于把跑了十几年的核心账务系统换过来，靠的是逐行业、逐系统地把稳定性证明出来，宝兰德在中移动磨了十几年才敢说"国内仅此一家"，就是这个过程的缩影。这不是技术一刀切的事，是信任一寸寸挪的事。

## 六、信创云平台：另一条替代线

中间件之上、应用之下，还有一层正在被替代的，云平台。准确说，是政企的私有云/专有云底座。

这一层过去也基本是国外货打底：VMware 的虚拟化、国外厂商的超融合。信创推开后，政企客户要的不只是"云"，而是"跑在国产 CPU、国产操作系统上、能过等保、能进采购名录的云"。于是冒出一批信创云/政务云厂商，格局跟前几层一样：政策驱动、国产替代。

格局大致几条线。一是大厂的专有云/私有云底座，阿里云、华为云都有政企版本（华为云 Stack 尤其在政务、金融落地多），它们的技术成熟、生态全，但国产化程度要分项目看，底座是否跑在鲲鹏等国产 CPU 上、是否过等保，因部署而异。二是央企背景的云，中国电子云（中国电子旗下，信创血统最正）、中国电信天翼云（国资云主力，政务云份额高）、移动云、联通云，这类天然贴合规需求。三是地方与行业云，浪潮云、曙光云等在政务、科研领域各有地盘。

这一层和中间件层的逻辑同源：替代的不是某个单一产品，而是"一整套底座"。VMware 替成国产超融合（深信服等也切这块），虚拟化层替成国产，云管平台国产，再往上是国产中间件、国产数据库，一层层摞起来，才是一个完整的信创云栈。中间件不过是这摞栈里的一环。理解了这一点就明白，为什么东方通、宝兰德的客户名单和云厂商高度重叠：大家服务的是同一批政企客户，在同一个信创项目里各管一层。

## 七、客观评价：替得扎实，却没什么存在感

回看中间件与云平台这一层，有个反差挺有意思：它替代得相当扎实，却在公众认知里几乎没什么存在感。

扎实，是因为中间件替代不靠口号，靠把核心系统跑通。中移动几亿用户的 BOSS 系统底下换成宝兰德，工行核心系统换上东方通——这种案例没有"差不多就行"的余地，能替进去就是真的能用。比起某些层"采购了但没真用"的现象，中间件这层的水分反而少。

没什么存在感，则由中间件的性质决定。它不像操作系统有桌面可看，不像办公软件人人会用，也不像 CPU 有参数可比。它是个后台管道，替得好用户无感，替得不好才上新闻——而这一层偏偏替得挺好，于是更没新闻。宝兰德替进中移动核心系统这种事，圈内当硬通货传，圈外几乎无人知晓。

代价也有。这层厂商高度依赖政企采购，民间市场几乎没有，一个普通创业公司不会去买东方通的中间件，它直接用云厂商的 PaaS 或开源 Tomcat 就完了。这意味着国产中间件的成长曲线绑死在信创采购的节奏上，采购旺则旺、采购缓则缓。宝兰德、普元往 AI、低代码延伸，某种程度上也是想跳出这个绑定，找到政企之外的第二个增长点。能不能跳出去，还在路上。

一句话收束：中间件与云平台在信创栈里替得扎实，却没挣到多少名声。东方通、宝兰德、普元这些厂商，把 WebSphere/WebLogic 和 VMware 底座一寸寸替出了政企核心系统，代价是它们的故事几乎只在政企圈里流传。
