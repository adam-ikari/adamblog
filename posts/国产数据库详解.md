# 国产数据库详解

## 前言

信创四大领域里，基础软件这一块有两条大腿：操作系统和数据库。前几篇讲完了 CPU、整机、操作系统，这一篇落到数据库。

数据库这一环在"去 IOE"叙事里分量极重。2014 年银监会提"去 IOE"，那个"O"就是 Oracle。把金融核心系统里的 Oracle 换掉，是国产数据库最硬的仗，也是衡量其含金量最直接的尺子。国产数据库不止一家，市场上叫得出名的有六家：达梦、人大金仓、神通、南大通用、OceanBase、GaussDB。它们来历不同、内核不同、定位不同，笼统说"国产数据库"会把根本差异抹平。本报告逐一拆解，并以一张表收束对比。文中事实尽量核至来源，核不到的明确标注。

需要先点明一个判断框架：**自主程度**。国产数据库的"自主"不是一个口径。有的从内核到代码全自研（OceanBase、GaussDB、达梦），有的是在 PostgreSQL 等开源基础上深度改造（人大金仓属此类），有的则在特定领域自研、其他借力。这条差异，决定了它们在信创语境下的自主程度排序，也决定了金融等关键行业选谁、不选谁。

## 一、达梦：最像 Oracle 的自主内核

达梦的全称是武汉达梦数据库有限公司，2000 年成立，总部在武汉。它的前身是华中科技大学数据库与多媒体研究所，是国内最早从事数据库管理系统研发的科研机构之一。创始人冯玉才，出身高校科研，这条学术血脉是达梦的底色。

达梦最被强调的标签是"完全自主知识产权"。它走的是自研内核路线，不基于 PostgreSQL、MySQL 等开源代码起家，这一点在国产数据库里相对少见，也是它在信创语境下分量重的关键，自主率高，过合规审计时底气足。维基条目对"完全自主"标注了来源请求，意味着该说法尚需进一步核实，但达梦走自研内核而非开源改造，这一点在行业内基本是共识。

达梦的定位，一句话概括是"最像 Oracle 的国产数据库"。它的 SQL 兼容性、PL/SQL 过程语言支持，都明显对标 Oracle，意在让原本跑 Oracle 的系统迁移过来时改造成本最低。这一策略很务实：金融、政企的核心系统多是 Oracle 写的，要替换，最省事的是找个"长得像"的。达梦在党政、金融、政务等关键行业落地较多，是信创数据库采购名录里的常客。它隶属中国电子信息产业集团（CEC），央企背景，与麒麟、长城同系，在 CEC 的信创闭环里占数据库这一环。

达梦的短板也来自它的出身。高校科研转商业，工程化与生态运营起步偏晚，相比互联网大厂孵化的 OceanBase、GaussDB，在分布式、云原生这些新架构上的节奏稍慢。它是"自主内核"的代表，但不是"技术最前沿"的代表。

## 二、人大金仓：基于 PostgreSQL 的务实派

人大金仓，现全称中电科金仓（北京）科技股份有限公司，简称"电科金仓"，1999 年成立，是成立最早的拥有自主知识产权的国产数据库企业之一，隶属中国电子科技集团（CETC）。名字里的"人大"来自它的出身，早年与中国人民大学有渊源，这一点在维基和官网的公司沿革里有所体现，不过其官网当前介绍主要强调 CETC 成员企业身份，人大背景未在官网突出，此处并录。

金仓的核心产品是 KingbaseES（KES），定位为企业级大型通用数据库。关于它的内核来源，行业内的共识是 KingbaseES 基于 PostgreSQL 深度改造而来——这是它和达梦最大的不同。达梦自研内核，金仓借力 PostgreSQL。这不丢人，PostgreSQL 是成熟的开源内核，在它之上做企业化增强是国际上也常见的路径，但严格说，这使金仓的"自主"口径与达梦有别：它更准确的说法是"基于开源内核的自主产品"，而非"从零自研"。官网称 KES V9 核心源代码自主率达 100%，这一数字是改造深度的体现，不改变其 PostgreSQL 出身。

金仓的市场表现相当扎实。官网引用赛迪顾问报告称，自 2020 年起连续五年在国产数据库关键应用领域销售套数占比第一，累计装机部署超 110 万套，2023、2024 年蝉联中国医疗与交通行业销量第一。这些数字虽为厂商引用的第三方数据，但金仓在党政、医疗、交通等行业的覆盖面，确实是六家里最广的之一。它的策略与达梦类似，靠适配与采购驱动铺量，而非靠技术前沿性取胜。

一个细节：金仓也兼容 Oracle，目标同样是承接 Oracle 迁移市场。它和达梦在这一块是直接竞品，差异主要在内核路线（开源改造 vs 自研）和所属央企（CETC vs CEC）。

## 三、神通：航天系的低调玩家

神通数据库，隶属神舟通用数据技术有限公司，背景是航天系。关于它的公开可核信息，远不如前几家丰富，维基无独立条目，其官网在本次查阅时反复重定向、未能稳定访问，赛迪等第三方报告里它的份额数据也较零散。因此本节以方向性描述为主，具体规格与份额以官方披露为准。

能确认的是：神通在国产数据库里属"航天系"一脉，与中国航天科技集团（CASC）的软件业务相关，产品线覆盖事务型与分析型数据库，长期服务于政务、军工、航天等场景。它的市场存在感不如达梦、金仓响，但凭借航天背景在特定领域有稳定采购。在六家国产数据库的对比里，神通更接近"有特定场景、但公开声量与生态较小"的一类，本文不展开未核实的细节。

## 四、南大通用：分析型的专才

南大通用，全称天津南大通用数据技术股份有限公司，"南大"指向其与南京大学的渊源（公司注册于天津，研发有南大背景）。它专注数据库约二十年，是国内分析型数据库市场的长期领跑者，官网称其蝉联中国分析型数据库市场第一份额十余年，并连年进入 Gartner 数据仓库魔力象限，这是六家里少有的国际口径认可。

南大通用的产品线以 GBase 品牌统称，当前主推几款：

- **GBase 8a**：面向海量数据查询分析的分析型数据库，用作数据仓库、BI、决策支持系统的承载，是南大通用的招牌。
- **GBase 8s**：基于共享存储的数据库集群，定位为企业级交易型（OLTP）数据库，支持国密算法，达国标第四级安全标准，厂商称能在 80% 以上场景替换国际主流数据库。
- **GBase 8c**：多模多态分布式数据库，兼容 Oracle、PostgreSQL、MySQL 协议，属较新的第三代产品线。

这里要提一句容易被搞混的地方：早年南大通用确有 GBase 8t（事务型）产品，但当前官网主推线里 8t 已不在列，事务型角色由 8s 承担。网络上不少资料仍提 8t，属历史产品名，与现状有出入，以官方当前产品线为准。

南大通用的特色是"专才"——在分析型/数据仓库这一块做得深，市场份额扎实。但它的短板也在此：在 OLTP 事务型核心系统、特别是金融核心替换 Oracle 这块硬骨头上，它的声量不及 OceanBase、GaussDB。它更像是"数据仓库领域的国产首选"，而非"全场景通吃"的选手。官网强调各产品"拥有自主知识产权"，但其分析型内核的历史血缘（早年与开源列存技术的关系）公开资料有限，此处不展开。

## 五、OceanBase：互联网大厂孵化的金融级标杆

OceanBase 是六家里技术叙事最完整、也最常被当作"国产数据库标杆"的一家。它的来历可精确到年：2010 年立项于阿里巴巴，由阳振坤主持研发。阳振坤的加入是项目的起点，他 2010 年入职阿里，OceanBase 随之立项。后来业务随组织调整归于蚂蚁集团（原蚂蚁金服），2020 年 6 月注册独立公司"北京奥星贝斯科技有限公司"运营。

OceanBase 诞生的直接动因，是淘宝与支付宝的交易量迈向天量，Oracle 集中式数据库扛不住、授权费也极贵。2009 年淘宝宣布放弃 Oracle，转投自研架构，OceanBase 即是这一决心的产物。它的关键技术节点，可拉一条时间线：

- 2011 年，0.1 版用于淘宝收藏夹。
- 2014 年，0.5 版引入 Paxos 共识协议，在支付宝交易系统上线，双十一负担 10% 流量，这是"金融级高可用"的起点。
- 2015 年，支付宝核心账务系统整体迁至 OceanBase，成为全球首个应用于金融核心业务系统的分布式关系数据库。
- 2016 年，1.0 版上线，支撑双十一 12 万笔/秒支付峰值，全面取代 Oracle。
- 2019 年，在 TPC-C 基准测试中登顶，每分钟 6088 万笔交易（60.88M tpmC），打破 Oracle 保持九年的纪录。
- 2020 年，性能分数突破亿级，达 7.07 亿 tpmC。

这条时间线最硬的事实是：**OceanBase 真的在金融核心系统里替换掉了 Oracle，而且扛过了双十一。** 这在国产数据库里是独一份的履历。达梦、金仓也兼容 Oracle、也接迁移市场，但在"支付宝核心账务这种极致场景下实证替代 Oracle"这一条上，OceanBase 的履历最硬。

技术上，OceanBase 是原生分布式关系数据库，采用 LSM-tree 存储引擎，支持 HTAP（OLTP 与 OLAP 混合负载），兼容 MySQL 与 Oracle 协议，有开源社区版与企业版。它的"自主"是全栈自研——从分布式架构到存储引擎，不基于某个开源数据库改造，这一点与达梦同属"自研内核"一档，但起点是互联网实战而非高校科研。

OceanBase 的定位偏金融与互联网核心系统，是替代 Oracle 最被看好的选手。它的短板在政企办公与党政场景——那一方面采购驱动、靠适配铺量的市场，OceanBase 的渗透不及金仓、达梦。它是"攻坚最难的场景"的尖兵，不是"铺最广的面"的主力。

## 六、GaussDB：华为的全栈协同

GaussDB 是华为的数据库产品，定位为企业级分布式、云原生、AI 原生关系数据库。它的历史比许多人想象的早：华为自 2001 年起即研发自研内存数据库，2011 年定名 GaussDB，2018 年组合为面向多行业的产品体系，2019 年 5 月 15 日在北京的开发者大会上正式发布，定位"AI 原生数据库"。2023 年 6 月推出新一代，强调从芯片、服务器、存储、网络、操作系统到数据库的全栈协同自主。

技术上，GaussDB 遵循 ACID，支持分布式与云原生部署，提供 MySQL 兼容版与 NoSQL 版，主要作为华为云的托管服务提供。2020 年 7 月，华为将开源版本 openGauss 捐给开放原子开源基金会，以 Apache 许可证开源，仓库托管于 Gitee——这一步与麒麟 openKylin 的开源化逻辑相似，都是"商业版专有 + 社区版开源换生态"的双轨。

GaussDB 最亮眼的市场成绩在金融。它支撑了工商银行、邮储银行、建设银行、农业银行等国有大行的核心系统，这是国产数据库在金融核心场景落地最硬的案例之一，与 OceanBase 形成双雄格局。它的优势是华为的全栈能力，鲲鹏 CPU + 欧拉操作系统 + GaussDB 数据库，软硬件协同自主，在信创全栈语境下这一条很有竞争力。

GaussDB 的"自主"与 OceanBase 同档，全栈自研，不基于开源数据库改造。它与 OceanBase 的差异更多在出身（华为通信/企业基因 vs 阿里互联网基因）与生态策略（华为云托管 + openGauss 开源 vs 蚂蚁独立公司 + 社区版开源）。两家在金融核心替代 Oracle 这块正面竞争，是国产数据库里技术叙事最强的两家。

## 七、六家对比：一张表收束

讲了六家，差异已铺开。用一张表收束，便于对照。

| 厂商 | 来历/归属 | 内核来源 | 定位与强项 | 信创适配/落地场景 |
|---|---|---|---|---|
| **达梦** | 武汉达梦，2000，CEC 旗下，冯玉才 | 自主内核（不基于开源） | 最像 Oracle，党政/金融迁移 | 党政、金融核心，采购名录常客 |
| **人大金仓** | 电科金仓，1999，CETC 旗下 | 基于 PostgreSQL 深度改造 | 通用型，铺量最广 | 党政、医疗、交通，销量套数领先 |
| **神通** | 神舟通用，航天系（CASC） | 公开信息有限，以官方为准 | 特定场景，声量较小 | 政务、军工、航天 |
| **南大通用** | 天津南大通用，约 2004 | 自主知识产权（分析型见长） | 分析型/数据仓库第一份额 | 数据仓库、BI、电信、党政 |
| **OceanBase** | 蚂蚁集团，2010 立项，阳振坤 | 全栈自研，原生分布式 | 金融级，实证替代 Oracle | 金融核心、互联网，双十一扛过 |
| **GaussDB** | 华为，2001 起研，2019 发布 | 全栈自研，云原生/AI 原生 | 金融核心，全栈协同 | 四大行核心系统，华为云 |

这张表能看出几条规律。

**自主程度分两档。** 自研内核一档：达梦、OceanBase、GaussDB——从内核代码起自研，不基于开源数据库改造，"自主"口径最硬。开源改造一档：人大金仓基于 PostgreSQL——借力成熟开源做企业化增强，自主率高但出身非自研。神通、南大通用的内核血缘公开资料有限，不强行归类，以官方为准。这条分档，在金融核心等最看重自主可控的场景里，直接决定选型优先级。

![国产数据库六家按内核来源分档](/posts/国产数据库详解/database-six-tiers.svg)

**出身决定基因。** 高校科研转商业（达梦）、央企嫡系铺量（金仓、神通）、互联网大厂攻坚（OceanBase、华为全栈）——三种出身，三种打法。高校系自主但工程化慢，央企系适配广但技术前沿性弱，大厂系技术强但政企渗透需补课。

**替代 Oracle 是分水岭。** 六家都说兼容 Oracle、都接迁移市场，但真正在金融核心系统里实证替换掉 Oracle 并扛过极致流量的，只有 OceanBase 与 GaussDB。达梦、金仓在党政和中等金融场景替代 Oracle 也有落地，但"支付宝核心账务""四大行核心"这种最硬的仗，是大厂系打的。这一条，是技术含金量的真实分界。

## 八、客观评价：自主是有代价的，但这一仗打得值

国产数据库这一层，在信创各层里进展最扎实、也最能拿出硬成绩：OceanBase 打破 Oracle 九年 TPC-C 纪录、四大行核心系统跑上国产数据库、支付宝双十一由自研分布式数据库扛住——这些不是宣传话术，是可核验的事实。在"去 IOE"提出十几年后，那个"O"确实被啃下了相当一块，这是国产数据库真实的功劳。

代价也摆在那里。达梦、金仓这类自主内核/开源改造的厂商，在分布式、云原生等新架构上节奏偏慢，面对 OceanBase、GaussDB 的技术代差，压力不小。而 OceanBase、GaussDB 技术虽强，在政企办公、党政这些靠适配铺量的场景里，渗透又不及金仓、达梦，技术先进不等于采购中标。两类厂商各擅胜场，但也各有够不到的地方。

还有一层不容回避的代价：自主内核的生态。自研内核意味着与国际开源生态（PostgreSQL、MySQL 的插件、工具、人才池）有一定距离，二次开发与人才供给都更窄。OceanBase、GaussDB 走开源（社区版/openGauss）正是在弥补这一点，用开源换生态、换人才。但开源社区的真正活跃度，与 PostgreSQL、MySQL 这类全球根社区相比，差距仍明显。自主是有代价的，这一代价在数据库领域同样成立。

最后一句：国产数据库是信创全栈里最接近"既自主又能打"的一环。它在最硬的金融核心场景交出了答卷，也在最广的政企采购场景铺开了量。两类厂商、两条路线并存，短期看互补大于竞争，长期看谁把"自主"和"生态"两头都做厚，谁才走得远。
